Damo Tracker

一个面向量化复盘的自动追踪任务：检索“大摩闭门会”视频，优先获取字幕，必要时转写音频，从明确推荐里提取 A 股/港股标的，并把覆盖率、截断风险和失败原因一起交代清楚。

查看 README 样例报告 
$ ./damo_run.sh
范围：昨天
扫描：47 个视频
已分析：40 个，占 85.1%
截断但已分析：10 个
完全失败：7 个
提取标的：10 支

输出：
- damo_result.json
- damo_report_current.md
- damo_runs/...
默认区间默认追踪前一天的视频与观点
模型无关交给当前 OpenClaw 环境路由
截断可用分析片段，同时显式标风险
可接入输出 JSON/Markdown，方便外部复盘系统消费

工作流

01

检索视频

用 OpenCLI 与 B 站接口双路径搜索“大摩/摩根士丹利”。

02

获取内容

优先字幕，失败后用 yt-dlp、Whisper、浏览器和 API 兜底。

03

提取标的

只提明确看好、买入、超配或可布局的 A 股/港股。

04

校验去重

兼容 ASR 错字和股票别名，保留 raw 候选供复核。

05

生成报告

输出覆盖概览、标的速览、详情、截断片段与失败诊断。

为什么不是只给股票列表

视频源经常遇到 B 站 412、无公开字幕、试看媒体或浏览器桥接不可用。报告会把扫描数、失败数、截断数、raw 候选和去重标的一起展示，避免把不完整结果误判成完整追踪。

可选的个人编排

每日追踪、周六 A 股周报、周日知识沉淀，是我的 OpenClaw 个性化配置示例。项目本体只负责追踪、提取、报告和归档，外部系统可以按自己的节奏消费结果。

Optional cron Personal weekly report Custom review flow

报告结构

  1. 覆盖概览：扫描、已分析、截断、失败、提取数量
  2. 标的速览：一行看完股票、评级、目标价和一句话逻辑
  3. 标的详情：理由、来源视频、发布日期和链接
  4. 截断片段：说明哪些内容只基于片段分析
  5. 失败诊断：把技术报错压缩成可行动的人话

失败诊断
- B站/yt-dlp 412 或反爬限制
- 视频没有可用公开字幕
研究观察工具，不构成投资建议。所有模型提取结果都应回到原视频和报告 JSON 复核。
